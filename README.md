# moral-mechinterp

`moral-mechinterp` is a reproducible first-stage research scaffold for behavioral evaluation of GT-HarmBench-style two-choice moral and game-theoretic prompts, using `uv`, Hugging Face inference, and paper-style summary plots. Given JSONL examples with `id`, `game_type`, `scenario`, `option_a`, `option_b`, and `safe_label`, it builds deterministic prompts ending in `Answer:`, scores each model's next-token preference for `" A"` versus `" B"`, infers the selected action, computes whether it is safe, records the safe-action logit margin `logit(safe) - logit(harmful)`, and constructs disagreement sets across Base (`unsloth/Qwen3.5-9B`), UT, and GAME trained models. The main entrypoints are `uv run python scripts/evaluate_behavior.py data/gtharmbench.jsonl --config configs/eval.yaml` and `uv run python scripts/summarize_behavior.py outputs/behavior/model_choices.csv`; outputs are written to `outputs/behavior/model_choices.csv`, `outputs/behavior/model_choices.jsonl`, `outputs/tables/`, and `outputs/figures/`. This stage intentionally focuses only on behavioral anchors for later mechanistic work such as CKA, logit lens, probes, activation patching, and moral/cooperative direction analysis.
